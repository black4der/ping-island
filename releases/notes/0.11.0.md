## 亮点
- Hook 安装现在可以按事件类别自定义，支持只接管审批、通知、生命周期或活动事件；也可以选择让工具审批和提问继续留在终端中处理。
- Claude Code / 兼容客户端的审批按钮现在严格跟随 bridge 暴露的 session-scope 能力，只有实际支持会话级审批时才显示 “Always Allow”，对应 issue `#151`。
- 设置页的权限状态会在窗口打开期间保持刷新，授权变化不需要关闭重开设置页才能看见。
- 集成配置文件夹入口的文案更短、更清晰，降低首次安装与排查 hook 配置时的理解成本。

## 修复
- 修复用户直接在终端批准 Claude Code 工具调用后，Island 侧仍可能停留在等待审批状态的问题。
- 修复 Codex usage 快照可能按文件修改时间选到旧 token_count 的问题，现在会按事件时间选择最新配额数据。
- 修正 OpenCode 插件生成路径，跟随当前官方配置目录。
- 保留 app-server 刷新后的 Codex 建议隐藏与审批介入状态，避免刷新覆盖仍待处理的 UI 操作。

## Highlights
- Hook installs can now be customized by event category, covering approvals, notifications, lifecycle events, and activity events; users can also leave tool approvals and questions in the terminal.
- Claude Code and compatible approval buttons now follow explicit bridge session-scope capability, so “Always Allow” only appears when the client actually supports session-level approval, matching issue `#151`.
- Settings permission status now stays live while the settings window is open, so permission changes show up without reopening the panel.
- Integration configuration folder actions now use shorter, clearer copy for install and troubleshooting flows.

## Fixes
- Fixed Island-side approval state that could remain stuck after a Claude Code tool call was approved directly in the terminal.
- Fixed Codex usage snapshots choosing stale token_count data by file modification time; freshness now comes from the event timestamp.
- Corrected OpenCode plugin generation to use the documented configuration directory.
- Preserved Codex suggestion hiding and pending approval intervention state after app-server refreshes.

## Notes
- The Sparkle build number is now `36`, and the app version is now `0.11.0`.
- This release focuses on hook install control, approval consistency, usage accuracy, and live settings state.
