## 修复
- 远程 SSH bridge 启动前会先确认远端二进制已安装，启动失败时会回传最近日志，方便定位连接问题。
- Linux 远程 bridge 下载会优先使用 `musl` 静态包，并兼容回退到旧命名的 release 资产，避免部分远程主机因资产命名差异无法完成安装。
- 远程集成卸载会跳过不存在的远端 hook 配置，不再为从未安装过的客户端创建空配置文件。
- 设置页打开辅助功能权限前会刷新 macOS TCC 信任状态，减少已授权后界面仍显示待授权的情况。
- 补齐远程集成说明的英文和简体中文本地化，避免英文界面露出中文源文案。

## 体验
- 调整 app icon 资源留白，降低 Dock、Finder 和 release notes 中图标视觉过满的问题。

## 说明
- Sparkle build 号提升到 `38`，版本号提升到 `0.11.2`。
- 这是基于 `0.11.1` 的远程连接与发布资产兼容性修复版本。

## Fixes
- Remote SSH bridge startup now checks that the remote binary is installed first and returns recent logs when startup fails, making connection failures easier to diagnose.
- Linux remote bridge downloads prefer the `musl` static package while falling back to legacy release asset names, avoiding install failures caused by asset naming differences.
- Remote integration uninstall now skips missing remote hook configs instead of creating empty config files for clients that were never installed.
- The settings panel refreshes macOS TCC Accessibility trust before opening permission settings, reducing stale pending-permission states after access has already been granted.
- Added missing English and Simplified Chinese localization for remote integration guidance so English UI no longer leaks the Chinese source string.

## Experience
- Adjusted app icon asset padding so the icon reads less oversized in the Dock, Finder, and release-note surfaces.

## Notes
- The Sparkle build number is now `38`, and the app version is now `0.11.2`.
- This is a remote connection and release asset compatibility patch based on `0.11.1`.
